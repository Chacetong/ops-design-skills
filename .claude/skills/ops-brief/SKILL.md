# ops-brief — 需求理解与结构化

将用户的模糊运营需求解析为标准化 brief JSON + 创意概念，供下游 Skill 使用。

## 输入

用户自然语言描述，例如：
> "做一个 Banner，主题是春季促销，标题是'春意盎然'，附带 product.png"

## 处理流程

### 1. 提取关键信息

从用户输入中识别：
- **主题**：核心内容方向（春季促销、新品上市、节日活动等）
- **标题文案**：需要在图片上呈现的文字（详见下方标题处理逻辑）
- **色调偏好**：如果用户提到了（"清新"、"热烈"、"高端"等）
- **附带素材**：用户是否提供了图片（本地文件或网络 URL），以及这些图片的用途
- **图片数量**：0 张 / 1 张 / 多张 — 决定底图来源（1 张用作参考图，其他情况由 AI 生成底图）

#### 网络图片自动下载

如果用户提供的图片是网络 URL（以 `http://` 或 `https://` 开头），在进入后续流程前先自动下载到本地：

```bash
curl -sL -o "<下载路径>" "<URL>"
```

- 下载路径：`/tmp/ops_ref_<序号>.<扩展名>`（从 URL 推断扩展名，默认 `.png`）
- 下载后用 `file` 命令验证是有效图片
- 后续流程统一使用下载后的本地路径，对下游透明

> 物料类型固定为 Banner（1920x960，单图直出，文字与视觉融为一体），无需向用户确认类型。

### 2. 标题处理逻辑

图片上的标题始终使用英文。根据用户输入的不同情况：

**情况 A：用户明确声明了标题**

以下表述视为「明确标题」：
- "标题是 X"、"title: X"、"用 X 做标题"
- 用户独立列出一个词/短语，语义上显然是作为 Banner 标题使用

以下**不视为「明确标题」**，应走情况 B 或 C：
- 角色名/IP名出现在内容或风格描述中（如"哪吒风格"、"画孙悟空"）
- 风格词/主题词出现在描述中（如"巴洛克"、"国风"、"极繁主义"）
- 活动名/品牌名夹在句子里未被明确声明为标题

→ 用户给出英文标题：直接使用，不做修改。
→ 用户给出中文标题：创作 3 个英文选项（见情况 B）。

**情况 B：用户提供了中文标题或中文主题（未给英文）**
→ 基于中文含义创作 3 个英文标题选项，供用户选择。要求：
- 简短有力（2-5 个单词为佳）
- 具有标题感和冲击力，适合运营物料
- 风格匹配主题调性（促销用激励语、品质用高端语、节日用氛围语）

示例：
> 用户输入："标题是'春意盎然'"
> 选项：
> 1. **Spring Awakens**
> 2. **Hello Spring**
> 3. **Bloom & Shine**
> 4. 给出 3 个新方案

使用 AskUserQuestion 让用户从 4 个选项中选择（用户也可以自行输入）。

**情况 C：用户没有提供任何标题（包括角色名/主题词仅出现在内容描述中的情况）**
→ 基于主题和参考图片的内容理解，创作 3 个英文标题选项供用户选择。如果用户提供了参考图片，仔细观察图片内容、色调、氛围来启发创作。

**不得将角色名、风格词、主题词自动升格为标题。** 必须通过 AskUserQuestion 让用户显式确认。

使用 AskUserQuestion 让用户从 3 个选项中选择（用户也可以自行输入）。

### 3. 生成创意概念

**这是整个流程的核心指导环节。** 根据提取到的所有信息（主题、标题、色调、参考图），产出一段完整的创意概念描述。后续所有环节（背景图、文字层）都必须严格遵循此概念。

创意概念应涵盖：
- **视觉主题**：画面核心要表达什么？（如"阳光穿过花丛洒在商品上的清新氛围"）
- **色彩方案**：主色、辅色、点缀色及其比例
- **画面构图**：主视觉元素的位置、文字与视觉的穿插关系、装饰元素分布
- **风格调性**：写实 / 扁平 / 插画 / 摄影风？温暖 / 冷冽 / 活泼 / 高端？
- **字体方向**：什么风格的字体最配这个概念（粗体无衬线 / 手写体 / 衬线等）
- **氛围关键词**：3-5 个英文关键词，作为后续 prompt 的锚点

#### 按图片数量分支处理：

**0 张图片（纯文字需求）：**
完全基于主题、标题、色调偏好进行创意构思。创意自由度最大，需要产出有视觉冲击力的概念。

**多张图片（≥2 张）：**
仔细观察所有参考图的共同元素、色调、风格，从中提炼统一的视觉方向。参考图作为风格/元素灵感源，不直接用于图生图。

**1 张图片：**
仔细观察图片的主体内容、色调、氛围。创意概念需围绕保留和延展这张图的视觉展开。特别注意：
- 描述图片主体是什么（产品、人物、场景等）
- 主体应保持在画面右侧
- 左侧需要自然延伸出适合放置文字的空间
- 延伸部分的色调、纹理应与原图和谐统一

#### 创意概念输出格式：

```
## 创意概念

**视觉主题**：<一句话描述画面核心>
**色彩方案**：主色 <色值/描述> | 辅色 <色值/描述> | 点缀 <色值/描述>
**画面构图**：<描述元素布局、视觉动线>
**风格调性**：<2-3 个关键词>
**字体方向**：<字体风格 + 原因>
**氛围关键词**：<3-5 个英文词，如 vibrant, fresh, warm glow, botanical>
```

### 4. 生成结构化 Brief

将理解后的需求 + 创意概念转化为标准化 brief JSON 并输出：

```json
{
  "type": "banner",
  "title": "Spring Awakens",
  "subtitle": "",
  "theme": "春季促销",
  "color_tone": "清新绿色系",
  "reference_images": [],
  "creative_concept": {
    "visual_theme": "阳光穿过樱花枝洒落的春日清新氛围",
    "color_scheme": {
      "primary": "soft pink #F9A8D4",
      "secondary": "fresh green #86EFAC",
      "accent": "warm gold #FCD34D"
    },
    "composition": "大标题左上角，樱花枝从右侧向左延伸与文字穿插，花瓣装饰散布全图",
    "style": "明亮、清新、自然摄影风",
    "typography_direction": "rounded modern sans-serif, friendly and inviting",
    "mood_keywords": ["fresh", "bloom", "warm glow", "spring breeze", "vibrant"]
  }
}
```

## 输出与确认

将创意概念和 brief JSON 一起展示给用户，使用 AskUserQuestion 交互确认：
- **确认** → 进入下一步
- **给出修改建议** → 融入建议重新生成 brief，再次确认

用户确认后，**自动继续后续流程**：
1. 调用 `/ops-prompt` 生成绘图 prompt 组
2. 调用 `/ops-gen` expand-only 生成扩图 → 用户确认
3. 调用 `/ops-gen` text-only 加文字 → 用户确认
4. 缩放为终稿 banner → 用户确认
